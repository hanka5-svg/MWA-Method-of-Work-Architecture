# CASES — Moduły analityczne MWA  
## Przypadki, ryzyka i architektury ochronne w pracy z LLM

Katalog **/cases** zawiera moduły analityczne, które pokazują, jak architektura MWA działa w praktyce — w sytuacjach ryzyka, nadużyć, konfliktów pól oraz w kontekstach wymagających ochrony zarówno **pola pracownika**, jak i **pola pracodawcy**.

To nie jest zbiór incydentów.  
To jest **mapa systemowych zjawisk**, które LLM może wykrywać, stabilizować i neutralizować.

---

# 1. Cel katalogu

Celem katalogu **/cases** jest:

- pokazanie realnych zagrożeń w środowisku pracy,  
- analiza nadużyć pracowniczych i menedżerskich,  
- opisanie mechanizmów wykrywania anomalii,  
- zdefiniowanie roli LLM jako strażnika pola pracodawcy,  
- jednoczesne zachowanie nienaruszalności pola pracownika,  
- integracja danych z Polski, USA i badań naukowych (CEJSH).

Każdy moduł jest samodzielny, ale razem tworzą **spójną architekturę ochronną**.

---

# 2. Logika katalogu

Moduły są ułożone od **najbardziej podstawowych** (nadużycia pracownicze) do **najbardziej zaawansowanych** (systemy wczesnego ostrzegania, governance).

Struktura:

1. **Moduły 7–8** — fundament dwupolowy i nadużycia pracownicze  
2. **Moduły 9–10** — strażnik kosztów i audyt zgodności  
3. **Moduł 11** — ryzyka kadrowe (PL + USA + CEJSH)  
4. **Moduł 12** — system wczesnego ostrzegania (EWS)  
5. (opcjonalnie) Moduł 13 — governance i nadzór właścicielski

---

# 3. Lista modułów

## **7. Dwupolowa architektura etyczna — pracownik vs. pracodawca**  
Opis dwóch pól:  
- pole pracownika (prywatność, dane osobiste),  
- pole pracodawcy (narzędzia, koszty, bezpieczeństwo).  
Fundament całej serii.

## **8. Nadużycia pracownicze — jak LLM może je wykrywać (BHP cyfrowe)**  
Mechanizmy wykrywania:  
- nadużyć,  
- marnotrawstwa,  
- nielegalnych działań pracownika.

## **9. LLM jako strażnik kosztów — świadczenia vs. nadużycia**  
Kluczowe rozróżnienie:  
- świadczenia pracownicze *za zgodą* (legalne),  
- przywłaszczenia *bez zgody* (nielegalne).

## **10. LLM jako audytor zgodności — naruszenia regulaminu pracy**  
Automatyczne wykrywanie:  
- naruszeń zasad,  
- niegospodarności,  
- wycieków danych.

## **11. LLM w HR — ryzyka kadrowe (PL + USA + CEJSH)**  
Najbardziej analityczny moduł:  
- porównanie nadużyć w Polsce i USA,  
- integracja danych naukowych (CEJSH),  
- tabela porównawcza,  
- wspólny mianownik: **czyn pracownika**.

## **12. LLM jako system wczesnego ostrzegania (EWS)**  
Predykcja ryzyk:  
- anomalie organizacyjne,  
- sygnały ostrzegawcze,  
- wczesne wykrywanie nadużyć.

---

# 4. Dlaczego te moduły są potrzebne

Bo nadużycia pracownicze i menedżerskie są:

- **systemowe**,  
- **powtarzalne**,  
- **kosztowne**,  
- **trudne do wykrycia**,  
- **zależne od słabości kontroli**,  
- **wspólne dla Polski i USA**.

Badania naukowe (CEJSH), dane operacyjne (USA 2024–2025) i dane polskie (2024–2025) pokazują jedno:

> **LLM nie jest zagrożeniem — LLM jest narzędziem ochrony przed realnymi zagrożeniami.**

---

# 5. Jak korzystać z katalogu

- Moduły można czytać osobno lub jako całość.  
- Każdy moduł zawiera:  
  - analizę,  
  - testy MWA,  
  - mapy przepływów,  
  - diagramy logiczne,  
  - przykłady,  
  - protokoły działania.

To jest **atlas ryzyk** i **architektura ochronna** dla organizacji pracujących z LLM.

---

# 6. Wspólny mianownik wszystkich modułów

> **Nadużycia — niezależnie od kraju — mają wspólne źródło: czyn pracownika naruszający pole pracodawcy.**

LLM w architekturze MWA:

- wykrywa,  
- stabilizuje,  
- chroni,  
- zapobiega,  
- nie narusza pola pracownika.

To jest fundament całego katalogu **/cases**.
